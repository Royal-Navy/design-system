# Colour Tokens

We use named colours (e.g. Neutral or Action) instead of colour literals (e.g. blue).
Colour palettes represent functions, so that we can make interactions and contexts easy
to distinguish.

## Base palettes

### Neutral
> For general application use, structure, data and information where we don't want to infer meaning via specific colour usage.

<DesignSystemPalette
  group="neutral"
  subtitle=""
/>

### Action
> To show an item is interactive.

<DesignSystemPalette
  group="action"
  subtitle="To show an item is interactive."
/>

### Success
> For positive action such as adding, or feedback. eg: "Data saved"

<DesignSystemPalette
  group="success"
  subtitle="Use the Success palette for positive feedback."
/>

### Warning
> To indicate an action or message that may produce undesirable results but not critical. eg: "Another user is editing this page"

<DesignSystemPalette
  group="warning"
  subtitle="Use the Warning palette to indicate an action or message that may produce undesirable results."
/>

### Danger
> For destructive actions such as deletion, or for a critical or blocking warning. eg: "Unable to reach server"

<DesignSystemPalette
  group="danger"
  subtitle="Use the Danger palette for destructive actions."
/>
